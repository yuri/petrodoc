Making Rockspecs, HTML and RSS from a single source

Overview · Installation · Using Petrodoc · Contact · License

Overview↑

Petrodoc allows you to use a single configuration file to generate a rockspec, HTML documentation (with Luadoc) and an RSS feed for new releases.

Installation↑

Petrodoc consists of a single file (petrodoc.lua). Here is a list of recent releases:

  • Petrodoc-10.12.28 - decoupled from Sputnik (Dec. 28, 2010)
  • Petrodoc-8.02.23 - the first version (Feb. 23, 2008)

You can also install Petrodoc as a LuaRock from the repository at http://sputnik.freewisdom.org/rocks/galaxy/.

Using Petrodoc↑

Petrodoc takes as input a single file that is similar to a LuaRocks rockspec but contains additional fields for generating documentation.

For example, to generate documentation for XSSFilter, we would use the following petrodoc file:

  • https://github.com/yuri/lua-xssfilter/blob/master/petrodoc

We would go into "lua-xssfilter" directory and run petrodoc.lua with "petrodoc" as the first argument.

This would give us:

  • doc/index.html (see XSSFilter)
  • a release file like xssfilter-10.12.28.tar.gz
  • xssfilter-10.12.28-0.rockspec (a rockspec)
